# Roadmap

## Phase 0: Foundation

Timeframe:

- weeks 1 to 3

Goals:

- align on target user and first use case
- complete discovery interviews
- decide whether MVP starts with friendship, dating, or hybrid connection
- finalize trust posture for onboarding

Deliverables:

- completed vision
- research synthesis
- MVP PRD
- architecture v1
- security and trust plan

## Phase 1: MVP Design And Platform Setup

Timeframe:

- weeks 4 to 8

Goals:

- design onboarding, mood check-in, and candidate discovery flows
- set up backend foundations and environments
- define initial trust score and moderation flows

Deliverables:

- product designs
- API contracts
- database schema
- CI/CD baseline
- staging environment

## Phase 2: MVP Build

Timeframe:

- weeks 9 to 16

Goals:

- build verified onboarding
- build profile, intent, and mood systems
- ship candidate retrieval and ranking v1
- ship mutual opt-in flow

Deliverables:

- alpha-ready backend
- client prototype
- trust rules v1
- analytics events
- QA plan

## Phase 3: Closed Alpha

Timeframe:

- weeks 17 to 20

Goals:

- test with a small verified cohort
- learn where recommendation quality fails
- measure safety friction versus trust benefit

Deliverables:

- alpha metrics review
- abuse and incident review
- ranking improvements
- refined onboarding copy and controls

## Phase 4: Beta

Timeframe:

- weeks 21 to 28

Goals:

- add safe messaging
- improve moderation tooling
- improve neighborhood and online recommendation balance

Deliverables:

- beta release
- moderation dashboard
- trust score v2
- launch playbook

## Phase 5: Public V1

Timeframe:

- after beta quality gates are passed

Goals:

- launch in one geography or one focused user segment first
- maintain trust quality while gradually increasing volume
- expand only when safety metrics remain healthy

Deliverables:

- production launch
- incident response runbook
- retention and satisfaction dashboard
- next-market or next-segment expansion plan
